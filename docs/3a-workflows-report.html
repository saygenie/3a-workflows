Section 01
Executive Summary
3A Program이 무엇이고, 이 플러그인은 왜 필요한가

3A Program (Agentic AI Acceleration)은 AWS SA와 고객 개발자가 3~5일 동안 production-grade AI 에이전트 애플리케이션을 함께 만드는 co-building 프로그램입니다.

3a-workflows는 이 co-building 기간 동안 Claude Code가 효과적인 코딩 파트너가 되도록 하는 메타 하네스(meta-harness)입니다. 에이전트를 만드는 과정 자체를 구조화하여, SA와 개발자가 검증된 방법론을 따라 일관된 품질의 결과물을 만들 수 있게 합니다.

Section 02
Problem & Solution
왜 이 플러그인이 필요한가
Without 3a-workflows
  • 매번 에이전트 설계를 백지에서 시작
  • 도구 설계 원칙이 프로젝트마다 다름
  • 평가 기준 없이 "잘 되는 것 같다"로 판단
  • Sprint 후 인수인계 문서 부족
  • SA의 노하우가 개인에게 귀속
With 3a-workflows
  • 4가지 핵심 산출물로 구조화된 시작
  • 7가지 설계 원칙이 워크플로우에 내장
  • Ground Truth 기반 정량 평가 (L2 Loop)
  • 자동 생성되는 최종 보고서와 인수인계 문서
  • SA 노하우가 플러그인으로 재사용 가능
Section 03
7 Design Principles
플러그인에 내장된 에이전트 설계 원칙
1
Harness Engineering
Agent = Model + Harness. Harness = Guides (feedforward) + Sensors (feedback). 빠른 검사를 먼저 하는 Quality Left 원칙.
2
Building Effective Agents
"가장 단순한 솔루션을 먼저 찾고, 복잡성은 결과가 개선될 때만 추가." 5가지 워크플로우 패턴의 점진적 복잡도.
3
Tool Design for Agents
통합 > 분할. 5요소 문서화. 에러 메시지를 프롬프트로. 시맨틱 식별자. Eval-driven development.
4
Context Engineering
컨텍스트 = 유한 자원. Right Altitude 원칙. Just-In-Time 로딩. Sub-agent 요약.
5
Enterprise Agent Best Practices
Day 1 Observability. 4가지 초기 산출물. Agent + Deterministic Code 경계. Crawl → Walk → Run.
6
AgentCore Evaluations
2-Tier 평가 (로컬 + AgentCore). 3-Level 평가 (Session → Trace → Tool). 13개 built-in evaluator.
7
Feedback Loops at Every Level
Generate → Evaluate → Improve 루프가 Feature(L1), Quality(L2), Sprint(L3) 3개 레벨에서 작동.

Section 06
Co-Building Workflow
SA와 개발자가 3~5일 동안 어떻게 일하는가
Day 1
/3a-kickoff — Foundation
요구사항 수집 → 4가지 핵심 산출물 생성 (Agent Definition, Persona, Tool Definitions, Ground Truth) → 아키텍처 결정 ("Start Simple") → Feature 목록 → Sprint 계획 → Observability 설정 → Commit
Day 2
/3a-feature × N — Core Build
핵심 기능 구현 (각 Feature마다 L1 Loop: 코드 → 테스트 → 수정 → 통과) → /3a-eval로 baseline 수립 → L2 Loop: 점수 미달 시 원인 분석 → 수정 → 재평가 → /3a-checkpoint
Day 3
/3a-review + /3a-eval — Quality
추가 Feature 구현 → /3a-review로 3-perspective 리뷰 (도구/컨텍스트/아키텍처) → Critical 이슈 수정 → /3a-eval로 개선 확인 → /3a-checkpoint
Day 4
/3a-deploy dev — Deployment
배포 전 체크 → AgentCore Runtime 배포 → Smoke test → Observability 확인 → 최종 평가 → /3a-checkpoint
Day 5
/3a-handoff — Handoff
Production 배포 → 최종 보고서 HTML 생성 (SVG 아키텍처 다이어그램 포함) → 인수인계 문서 → Crawl → Walk → Run 로드맵 → .3a/ 정리 → Final Commit

Section 08
4 Key Deliverables
Day 1에 반드시 만드는 핵심 산출물
1. Agent Definition

에이전트가 해야 할 것과 하지 말아야 할 것을 명확히 정의. MUST DO / MUST NOT DO / Scope / Success Criteria / Error Handling.

2. Agent Persona

톤, 성격, 언어 선호도, 경계 밖 행동. 사용자와의 커뮤니케이션 스타일을 체계적으로 정의.

3. Tool Definitions

각 도구를 5요소로 정의: Name, Parameters, Return Format, Error Conditions, Usage Guidance. 통합 원칙 적용.

4. Ground Truth

10~20개 초기 테스트 케이스. 5가지 카테고리: happy-path, edge-case, refusal, multi-tool, parameter-extraction.

Section 09
2-Tier Evaluation
에이전트 품질을 정량적으로 측정하는 평가 체계
Tier 1 — Local (Default)

외부 의존성 없이 어디서든 동작. Claude가 LLM-as-Judge로 4개 차원 평가:

  • Tool Selection (0-100)
  • Parameter Accuracy (0-100)
  • Refusal Accuracy (pass/fail)
  • Response Quality (0-100)
Tier 2 — AgentCore (Available)

AgentCore Evaluations API 활용:

  • 13개 built-in evaluator
  • 3-Level 평가 (Session/Trace/Tool)
  • Online evaluation (배포 후 지속)
  • On-demand + continuous

L2 Quality Loop — Root Cause Analysis

Failure TypeRoot CauseFix Target
Wrong tool selected도구 설명이 불명확하거나 중복tool-definitions.md — when-to-use 명확화
Wrong parameters파라미터 문서화 부족Tool parameter descriptions
Missed refusal가드레일이 너무 느슨agent-definition.md — MUST NOT DO
False refusal가드레일이 너무 엄격agent-definition.md — scope 확장
Low quality response시스템 프롬프트 가이드 부족System prompt 또는 few-shot examples

Section 11
Crawl → Walk → Run
Sprint 이후 고객의 자립 경로
Crawl — Sprint 결과물 (현재)
  • 핵심 기능이 동작하는 에이전트
  • Tier 1 평가 baseline 수립
  • 기본 Observability (logs + traces)
  • 수동 배포 프로세스
  • 30~50개 Ground Truth 테스트 케이스
Walk — 1~2주 후
  • 백로그 Feature 추가 구현
  • CI/CD 파이프라인 구축
  • Ground Truth 100개+ 확장
  • 모니터링 대시보드 + 알림 설정
  • Tier 2 평가 연동 (AgentCore)
Run — 1~2개월 후
  • Production auto-scaling
  • 지속적 Online evaluation + drift detection
  • A/B testing framework
  • 보안 강화 완료
  • 팀이 독립적으로 반복 개선 가능
